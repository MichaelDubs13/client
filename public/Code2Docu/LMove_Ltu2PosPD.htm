Functional description

LMove_Ltu2PosPD serves as the primary controller for the conveyor lift transfer unit (LTU).

The LTU is designed to interface with a maximum of four conveyors, each attached to one of its four sides. Please note that this function block does not support configurations exceeding four sides.

The external PLC logic and HMI should interface with the LTU function block rather than directly with the hardware. This block processes inputs and program commands from external ladder logic, actuating the turn unit, stopper, and cushions as necessary, and returning status outputs utilized by the external ladder logic. It also processes operator commands directly triggered from the HMI interface, providing on-screen display of status information.

The LTU function block is designated for the following core functions:

  1. Receiving an incoming pallet from up to three sides connected to an incoming conveyor.
  2. Sending a pallet to one of up to three sides linked to an outgoing conveyor.
  3. Enabling manual operator control via HMI.
  4. Facilitating fault handling and displaying alarms on the HMI.

PLC Program Integration

Each LTU requires a unique instance of LMove_Ltu2PosPD.

Moreover, external SCL logic is essential for configuring the LTU function. This is achieved through the use of the interfaceParameter In/Out tag.

HMI Components

Wincc Faceplate

A basic faceplate Ltu2Pos is included to visualize the general state of the LTU.

For a more comprehensive view, an additional faceplate, MoveLineHmiCtrl_Ltu2Pos, can complement the basic faceplate. Ltu2Pos is usually implemented with pop-up event that integrates the second faceplate, offering a detailed portrayal of LTU status. Moreover, it enables manual manipulation of the LTU when operating in manual mode.
